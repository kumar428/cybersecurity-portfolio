# Ethical Hacking – Fundamentals

## Objective
To understand the core concepts of hacking, ethical hacking, their differences, scope of learning, and the five phases involved in an ethical hacking process.

---

## 1. What is Hacking?

Hacking refers to the process of identifying and exploiting vulnerabilities in a computer system, network, or application to gain unauthorized access.

Hacking can be categorized as:
- Malicious (illegal and harmful)
- Authorized (for testing and security purposes)

---

## 2. What is Ethical Hacking?

Ethical hacking is the practice of legally testing systems, networks, or applications to identify security vulnerabilities. It is performed with proper authorization to strengthen the security posture of an organization.

The primary objective of ethical hacking is:
- To identify weaknesses before malicious attackers do
- To improve system security and resilience

---

## 3. Difference Between Hacking and Ethical Hacking

| Hacking | Ethical Hacking |
|--------|----------------|
| Unauthorized | Authorized |
| Illegal | Legal |
| Malicious intent | Security-focused |
| Data theft or damage | Vulnerability identification and mitigation |

---

## 4. Learning Scope of Ethical Hacking

Ethical hacking involves a wide range of technical domains, including:

- Networking fundamentals (TCP/IP, ports, protocols)
- Operating systems (Linux and Windows)
- Web application security
- Vulnerability assessment and penetration testing
- Security tools and frameworks

Commonly used tools include:
- Nmap (network scanning)
- Burp Suite (web application testing)
- Wireshark (packet analysis)

Career opportunities:
- Cybersecurity Analyst
- Penetration Tester
- Security Engineer
- Bug Bounty Hunter

---

## 5. Phases of Ethical Hacking

Ethical hacking follows a structured methodology consisting of five key phases:

### 5.1 Reconnaissance (Information Gathering)
This phase involves collecting information about the target system, such as domain details, IP addresses, and infrastructure.

### 5.2 Scanning
In this phase, the attacker identifies open ports, services, and potential vulnerabilities using various tools and techniques.

### 5.3 Gaining Access
The attacker exploits identified vulnerabilities to gain access to the system or application.

### 5.4 Maintaining Access
After gaining access, techniques are used to maintain persistence within the system for continued analysis or testing.

### 5.5 Covering Tracks
This phase involves removing evidence of the attack to avoid detection, simulating real-world attacker behavior.

---

## 6. Practical Example

In a real-world web application security test:
- The tester first gathers information about the target (Reconnaissance)
- Scans for open ports and vulnerabilities (Scanning)
- Exploits a vulnerability such as SQL Injection (Gaining Access)
- Maintains access for further testing (Maintaining Access)
- Finally removes traces of activity (Covering Tracks)

---

## Key Takeaways

- Understanding the five phases is fundamental for any cybersecurity role
- Practical application and documentation are essential for skill development
